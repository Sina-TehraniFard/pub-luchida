Luchida — アーキテクチャマップ

依存の方向: 外側 → 内側 に依存する。内側は外側を知らない。

Domain domain/

最内核。値オブジェクト、エンティティ、売買ルール、コマンド。外部依存なし。

rule/
売買判断ルール。エントリーと決済の判定
EntryRule.tsエントリー判定を実装するクラスのインターフェース
ExitRule.ts決済判定を実装するクラスのインターフェース
rule/sma-cross/
SMAクロス戦略
SmaCrossEntryRule.tsゴールデン/デッドクロスでエントリー判定
SmaCrossExitRule.ts逆クロスで決済判定
SmaCrossSignal.ts2本のSMAがクロスしたか検出
rule/shared/
戦略をまたいで共用するルール
FixedStopLossExitRule.ts固定pipsで損切りする決済ルール
command/
Ruleの判定結果を命令として表現
EntryCommand.tsエントリー命令（通貨ペア、売買方向、ロット）
ExitCommand.ts決済命令（対象ポジション、理由）
DoNothing.tsエントリー/決済条件未成立の判定結果
EntryReason.tsなぜエントリーしたかの理由テキスト
ExitReason.tsなぜ決済したかの理由テキスト
market/
市場データの値オブジェクト群
BuySell.ts「買い」か「売り」の2択
CurrencyPair.ts通貨ペア（USD/JPY等）
Price.ts価格（小数点以下の精度を保証）
Pips.ts値幅（何pips動いたか）
Spread.tsbid/askの価格差
TimeFrame.ts時間足の種類（1分/15分/1時間/日足）
Timestamp.ts時刻（ミリ秒精度）
ConvictionScore.tsシグナルの確信度（0〜1）
EntryResult.ts注文が通った結果（約定価格等）
ExitResult.ts決済が通った結果（損益等）
TimeFrameBook.ts複数時間足のデータをまとめて管理
market/candle/
tickからローソク足を組み立てる
CandleAccumulator.tstickを溜めてローソク足を組み立てる
ConfirmedCandle.ts確定済みローソク足（もう変わらない）
FormingCandle.ts形成中ローソク足（まだ動いている）
CandleEvent.ts足が更新/確定したイベント
CandleOpenTime.ts足の始まり時刻
CandleCloseTime.ts足の終わり時刻
market/indicator/
テクニカル指標の計算と保持
IndicatorLedger.tsSMA等の指標を足確定ごとに計算・記録
IndicatorConfig.ts指標の設定（SMAの期間等）
IndicatorValues.ts計算済み指標値のセット
SmaCalculator.tsSMA計算を実装するクラスのインターフェース
SmaSnapshot.tsSMA(20)とSMA(100)の現在値セット
SmaValue.tsSMAの値オブジェクト
market/snapshot/
ある時点の市場データ一式
MarketSnapshot.ts全時間足の現在値をまとめたスナップショット
TimeFrameSnapshot.ts1つの時間足のtick+足+指標
market/tick/
リアルタイム価格
Tick.ts最新のbid/ask価格
TickTimestamp.tstick受信時刻
position/
ポジション管理
Position.ts保有中/決済済みのポジション（エンティティ）
PositionId.tsポジションの識別子
Lot.ts取引数量（1,000通貨単位）
OpenPositions.ts保有中ポジションの一覧
error/
ドメインエラー
BrokerError.ts注文失敗時のエラー
MarketDataError.ts市場データ取得失敗時のエラー
↑ Port は Domain の型のみ使用
Port port/

外部接続のソースコードを実装するクラスのインターフェース。

インターフェースにする理由: テスト時にモックに差し替えられる。本番はGMO、テストはスタブ。Rule や Action が特定のAPI実装に依存しなくなる。

Broker.ts注文発注・決済を実装するクラスのインターフェース
MarketDataPort.tstick受信を実装するクラスのインターフェース
MarketDataStreamPort.ts市場データ配信の開始/停止を実装するクラスのインターフェース
CandleHistoryPort.ts過去足取得を実装するクラスのインターフェース
PositionRepository.tsポジションのDB永続化を実装するクラスのインターフェース
UiNotifier.tsフロントエンド通知を実装するクラスのインターフェース
↑ Action は Domain + Port に依存
Action action/

Portを使って1つの業務操作を実行する。注文を出す、決済する、ポジションを記録する。

EntryExecution.tsBrokerに注文を出し、PositionをDBに保存
ExitExecution.tsBrokerに決済を出し、Positionを更新
↑ Application は Action + Port に依存
Application application/

市場データ受信→Rule評価→Action実行の全体制御。ビジネスロジックは持たない。

TradingSession.ts起動→市場監視→Rule評価→エントリー通知/自動決済の全体制御
↑ Adapter は Port を実装する
Adapter adapter/

Portのインターフェースを実装する。GMO APIの言葉を内部の言葉に翻訳する。

gmo/GMO コイン外国為替 FX API の実装
  GmoBrokerAdapter.tsGMO APIで注文を出す（Broker実装）
  GmoMarketDataAdapter.tsGMO WebSocketからtickを受信（MarketDataPort実装）
  GmoCandleHistoryAdapter.tsGMO APIから過去足を取得（CandleHistoryPort実装）
  GmoRestClient.tsGMO REST APIとのHTTP通信処理
  GmoWebSocketClient.tsGMO WebSocketの接続・再接続管理
  GmoApiError.tsGMO API固有のエラー定義
indicator/テクニカル指標ライブラリの実装
  TradingSignalsSmaCalculator.tstrading-signalsライブラリでSMAを計算（SmaCalculator実装）
↑ Infrastructure は Adapter + Port に依存
Infrastructure infrastructure/

データベース、サーバー、ログ。技術基盤そのもの。ドメイン非依存。

MarketDataStream.tstickを受け取り→足組立→指標更新→snapshotを通知
database/PostgreSQL
  connection.tsDB接続を生成する
  PostgresPositionRepository.tsポジションをDBに保存/取得（PositionRepository実装）
  schema/テーブル定義（Drizzle ORM）
    positions.tspositionsテーブルの定義
    positionEntrySnapshots.tsエントリー時の指標値を記録するテーブル
logging/ログ
  Logger.tsログ出力（レベル制御付き）
server/HTTPサーバー
  ExpressServer.tsREST API + Socket.IO サーバー
  SocketIoUiNotifier.tsWebSocketでフロントに通知（UiNotifier実装）
↑ main.ts が全てを組み立てる
Composition Root src/

全レイヤーの依存関係を組み立てて起動する。全レイヤーへの依存を持つ唯一の場所。

main.ts全クラスを生成し、依存を注入して起動する
index.tsエントリーポイント（main.tsを呼ぶだけ）
